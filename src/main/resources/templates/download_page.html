Reaper script Note names

Internet connection required

    The script for Reaper is quite basic. All processing and data are handled on the server. The script communicates with the main application over the internet, so a reliable internet connection is required.

Installation

  • In Reaper, click to Options -> Show Reaper resource path in explorer/finder.
  • The "satbify.lua" file should be placed inside the "Scripts" folder.
  • The "satbify_note_names.txt" file should be placed inside the "MIDINoteNames" folder.
  • When moved, in Reaper's Actions list, add the script, and assign any shortcut.
  • Open the MIDI editor, go to File -> Note/CC name -> Load note/CC name from file -> Choose file... -> select "satbify_note_names.txt" file.

What's New in Version 0.03

Important:

  • Satbify now only works within the time selection: note start must be within the time selection.
  • The engine has been updated to ensure more consistent and predictable chord transitions.
  • On the voices tracks, use the C1(24) keyswitch if you want to harmonize specific notes within a voice. Otherwise, all the notes will be deleted.
  • Muted notes are ignored.

Less Important:

  • Satbify tracks can now be positioned anywhere in the project (no need to make them the first 5 tracks anymore).
  • Required tracks are now automatically created and named if they are missing.
  • Satbify tracks must contain one of the following in their names: 'main_satbify', 'soprano_satbify', 'alto_satbify', 'tenor_satbify', 'bass_satbify'.
  • If no track contains one of the required names, new tracks will be added and named accordingly.
  • If absent, new items will be created, or existing items will be resized to cover the time selection automatically.
  • Multiple items can now exist on the 'main_satbify' track within the time selection (on other tracks, items are still limited to one).
  • Fixed: Notes are inserted in global time regardless of the item's position.
  • Fixed: Single chord works.
